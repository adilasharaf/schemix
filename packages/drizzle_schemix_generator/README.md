# drizzle_schemix_generator

---

## Installation

```yaml
# pubspec.yaml
dependencies:
  schemix: any

dev_dependencies:
  schemix_builder: any
  drizzle_schemix_generator: any
  build_runner: ^2.15.0
```

### npm Dependencies

For the generated `.drizzle.ts` files to compile and run in your TypeScript environment, ensure you install the following npm packages:

```bash
npm install drizzle-orm uuid
npm install -D @types/uuid
```

## Quick Start

```dart
import 'package:schemix/schemix.dart';

@Schemix(
  tableName: 'posts',
  enableTimestamps: true,
)
class Post {
  @PrimaryKey(autoGenerate: true)
  final String id;

  final String title;

  @BelongsTo(User)
  final String userId;

  const Post({required this.id, required this.title, required this.userId});
}
```

```bash
dart run build_runner build
```

Produces `gen/post.drizzle.ts`:

```typescript
// AUTO-GENERATED BY SCHEMIX — DO NOT EDIT.
// Source: lib/post.dart

import { pgTable, text, timestamp, relations } from "drizzle-orm/pg-core";
import { v4 as uuidv4 } from "uuid";
import { users } from "./user.drizzle";

export const posts = pgTable("posts", {
  id: text("id")
    .$defaultFn(() => uuidv4())
    .primaryKey(),
  title: text("title").notNull(),
  userId: text("user_id").notNull(),
  createdAt: timestamp("created_at").defaultNow().notNull(),
  updatedAt: timestamp("updated_at").defaultNow().notNull(),
});

export const postsRelations = relations(posts, ({ one }) => ({
  userId: one(users, { fields: [posts.userId], references: [users.id] }),
}));

export type Post = typeof posts.$inferSelect;
export type NewPost = typeof posts.$inferInsert;
```

## Configuration

Override the default `models_package` option in your `build.yaml` if your model classes are exported from a differently-named package:

```yaml
# build.yaml (in your app package)
targets:
  $default:
    builders:
      drizzle_schemix_generator|drizzleSchemixBuilder:
        options:
          models_package: "my_app_models"
```

The `zodBuilder` also accepts a `dateTimeAsString` option (default `true`) via `zod_schemix_generator` — this has no effect here, but is noted for completeness when using both generators together.

## What Gets Generated

| Feature                                      | Trigger                                                           |
| -------------------------------------------- | ----------------------------------------------------------------- |
| `text()` column                              | `String` field                                                    |
| `integer()` column                           | `int` field (non-autoIncrement)                                   |
| `serial()` column                            | `int` field with `@AutoIncrement`                                 |
| `real()` column                              | `double` / `num` field                                            |
| `boolean()` column                           | `bool` field                                                      |
| `timestamp()` column                         | `DateTime` field                                                  |
| `jsonb()` column                             | `List<*>`, `Map<*,*>`, `@Embedded`, or unresolvable type          |
| `text()` FK column                           | `@BelongsTo` relation field                                       |
| `text()` with enum values                    | Enum field (`{ enum: statusValues }`)                             |
| `.$defaultFn(() => uuidv4()).primaryKey()`   | `@PrimaryKey(autoGenerate: true)` on a `String` field             |
| `.primaryKey()`                              | `@PrimaryKey` on any other field                                  |
| `.defaultNow()`                              | `@CreatedAt` or `@UpdatedAt` annotated field                      |
| Auto-injected `createdAt` / `updatedAt`      | `@Schemix(enableTimestamps: true)` (only if not declared)         |
| Auto-injected `deletedAt`                    | `@Schemix(enableSoftDelete: true)` (only if not declared)         |
| `.unique()`                                  | `@Unique` or `@SchemixField(unique: true)`                        |
| `.notNull()`                                 | Non-nullable, non-PK field                                        |
| `relations()` block                          | Any field with `@BelongsTo`, `@HasOne`, `@HasMany`, `@ManyToMany` |
| `one()` with fields + references             | `@BelongsTo` or `@HasOne` with `relationFieldName`                |
| `many()`                                     | `@HasMany` or `@ManyToMany`                                       |
| `TODO` comment in relations block            | `@HasOne` without `relationFieldName` — requires manual fix       |
| `import { v4 as uuidv4 } from 'uuid'`        | Any class with a UUID auto-generate PK                            |
| `import { relations } from 'drizzle-orm/…'`  | Any class with at least one relation field                        |
| Cross-file import for related table          | Relation target is defined in a different source file             |
| Inline enum `const` array + type alias       | Dart enum referenced by any field in the file                     |
| `$inferSelect` / `$inferInsert` type exports | Every generated `pgTable`                                         |

## What Gets Skipped

### Fields

| Condition                                         | Reason                                                    |
| ------------------------------------------------- | --------------------------------------------------------- |
| `@IgnoreField`                                    | Excluded from all outputs                                 |
| `@DriftIgnore`                                    | Local-only exclusion; treated as equivalent for Drizzle   |
| `@DrizzleIgnore`                                  | Explicit Drizzle-only exclusion                           |
| `@CloudOnly`                                      | Server-only field; no physical column needed              |
| `@HasMany`, `@HasOne`, `@ManyToMany` (non-owning) | No column emitted — relation is declared in `relations()` |

### Classes

| Condition                |
| ------------------------ |
| `abstractSchema == true` |
| `embeddable == true`     |
| `isEnum == true`         |

## Package Structure

```
lib/
├── drizzle_schemix_generator.dart   ← public barrel (DrizzleSchemixGenerator)
├── builder.dart                     ← builder factory; registers DrizzleSchemixGenerator, returns _NoOpBuilder
└── src/
    ├── generator.dart               ← DrizzleSchemixGenerator implements SchemixGenerator
    ├── column_builder.dart          ← DrizzleColumnBuilder: FieldInfo → Drizzle column expression
    ├── table_builder.dart           ← DrizzleTableBuilder: ClassInfo → pgTable block lines
    ├── relations_builder.dart       ← DrizzleRelationsBuilder: ClassInfo → relations() block lines
    ├── file_assembler.dart          ← DrizzleFileAssembler: assembles complete .drizzle.ts content
    ├── import_collector.dart        ← DrizzleImportCollector: collects pg-core and cross-file imports
    └── utils.dart                   ← snakeCase / camelCase / tableVarName extensions; skipField()
```

`lib/drizzle_schemix_generator.dart` exports only `DrizzleSchemixGenerator`. `lib/builder.dart` is the entry point used by `build.yaml`. Everything under `src/` is internal.

## How It Fits Into the Build Pipeline

This generator uses a no-op builder pattern: `drizzleSchemixBuilder` registers `DrizzleSchemixGenerator` with `GeneratorRegistry` so that `schemix_builder`'s file builder can dispatch to it, then returns a `_NoOpBuilder` that declares no outputs. The actual `.drizzle.ts` file is written by `schemix_builder|schemix_file`.

```
Phase 1 — schemix_builder|schemix_scan
  Reads lib/**.dart, writes lib/schemix_registry.json

Phase 2 — drizzle_schemix_generator|drizzleSchemixBuilder  ← runs before schemix_file
  No-op; registers DrizzleSchemixGenerator in GeneratorRegistry

Phase 2 — schemix_builder|schemix_file
  Loads registry, dispatches to DrizzleSchemixGenerator (id='drizzle')
  Writes gen/{name}.drizzle.ts
```

`DrizzleSchemixGenerator` never reads `schemix_registry.json` directly. The resolved `TypeGraph` is provided through `GeneratorContext` by `schemix_builder`.

## Rules This Package Must Never Break

- Does not depend on any other Schemix generator package.
- Does not depend on `schemix_builder` at runtime — only in `dev_dependencies`.
- Does not import private `src/` paths from `schemix` or `schemix_builder`.
- `skipField()` must remain the single authoritative skip-gate for all field-level exclusion decisions.
- `DrizzleColumnBuilder.buildColumn` and `columnFunctionsFor` must stay consistent — every column type returned by `buildColumn` must have its import function covered by `columnFunctionsFor`.
- `@HasOne` without `relationFieldName` must always emit a `// TODO` comment rather than silently emitting broken TypeScript.
